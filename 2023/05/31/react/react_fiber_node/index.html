React Fiber

React v16以前，ReactElement被称为React DOM，其Diff、更新、渲染都是同步的，此时由于JS引擎线程被占用，那么响应用户事件所要调用的回调函数就得不到执行，观感上就是UI的卡顿。

为了解决该问题，引入了Fiber架构，将React DOM拆分为Fiber Node，通过scheduler调度每个Fiber Node的构建、Diff等等，scheduler每执行若干任务就将控制权交还浏览器，原本同步的、完整的一个任务被切分成一个个子任务，类似于操作系统中时间分片的概念。

可见这是一个协调调度的过程，称为reconcile，分为两步：render和commit。

Fiber Node详解

作为树结构，它有三种基础属性：

  1. return指向其父节点；
  2. sibling指向其兄弟节点；
  3. child指向其首个子节点；

Fiber Node是以ReactElement为基础构建的，又是最小任务单元，因此还有一些其他重要属性：

  1. tag，标识组件类型；
  2. stateNode，指向对应的浏览器DOM节点或者组件实例（ClassComponent）；
  3. type，FunctionComponent指向对应函数，ClassComponent指向对应类，HostComponent存对应的HTML标签；

Diff算法相关：

  1. flags，存二进制数，用于记录Diff后对应的浏览器DOM需要做什么改变，取值Placement代表新增或替换，Deletion代表删除（flags |= Deletion增，flags &Deletion 验，flags &= ~Deletion删）；
  2. deletions，Diff后发现要删除的Fiber节点；

Effect List相关：

  1. effectTag，副作用类型；
  2. nextEffect，DFS顺序下，下一个有副作用的Fiber节点，构成一个单链表Effect List；
  3. firstEffect，Effect List的头节点；
  4. lastEffect，Effect List的尾节点；

调度优先级：

  1. lane，也是二进制数，表示Fiber Node的处理优先级，低优先级任务可能被打断，然后在下一次Reconcile处理；

知识点扩展（树的孩子-兄弟表示法）

React Fiber Tree实现了孩子-兄弟表示法，将树转化为二叉树，左孩子就是首个子节点，右孩子就是首个兄弟节点，然后Fiber Tree的构建和Commit顺序就可以看作是二叉树的先根（序）遍历。
